# cheatsheet-mk

本技能是一个完全独立的、自包含的速查表生成工具。它内置了内容组织规范、语法约束及元数据标准，确保产出的源文件具有高权威性、高可读性且兼容特定 HTML 渲染器。

## 🎯 核心目标与边界

- **产出物三件套**：
  1. `<topic>.md`：遵循特定子集语法的速查表核心内容。
  2. `meta.yml`：速查表的索引元数据。
  3. `refmap.md`：结构化的参考资源映射。
- **职责禁令**：禁止在本技能内生成或落盘 `<topic>.html` 文件。
- **独立性**：本技能不依赖任何外部本地规则文件，所有规则已内化。

## 🧠 生成策略与方法论

### 1. 资料采集策略 (Research Strategy)
- **权威优先**：采集顺序为：官方文档(Docs) > GitHub README/Wiki > 官方 Release Note > 社区高质量教程。
- **网址深度分析**：若输入是网址，必须使用浏览器工具读取页面，同时分析其关联的导航链接（如 "Getting Started", "API Reference", "CLI"），确保提取核心 API 且不使用过时/废弃的方法。
- **版本对齐**：记录整理时参考的具体版本号，将其写入 `meta.yml` 的 `version` 字段。

### 2. 内容编排策略 (Authoring Strategy)
- **模块颗粒度**：覆盖 8-15 个核心功能模块。
- **高频优先**：按“安装/基础 -> 核心命令/API -> 进阶技巧 -> 常见配置 -> 陷阱/FAQ”组织。
- **简洁性原则**：使用简体中文。以“如何做”为导向，避免长篇大论，多用短语和代码示例。
- **自包含示例**：每个代码块应尽量完整，带有必要的中文注释。

### 3. 重建与改进策略 (Refactor Mode)
- **增量优化**：在重建任务中，优先补齐缺失的核心模块，其次优化已有模块的排版和代码示例。
- **自检规则**：
  - 审视内容是否包含指定工具的全部核心功能。
  - 检查代码语言标识（lang）是否准确。
  - 确认 `meta.yml` 包含 `desc/tags/version/github/date` 五个必选字段。

## 📝 `<topic>.md` 语法规范（关键：兼容渲染器）

为了确保生成的 Markdown 能被后续工具正确解析为瀑布流卡片布局，必须严格遵守以下子集语法：

### 1. 文档头 (Frontmatter)
必须在文件开头包含 YAML 格式的 Frontmatter：
```yaml
---
title: Topic 速查
lang: bash
version: "1.0"
date: YYYY-MM-DD
github: owner/repo
---
```
- `lang`：文档默认代码语言。符合 `^[a-z0-9_+-]+$`。
- `github`：上游仓库路径，格式 `owner/repo`，不确定填 `unknown`。

### 2. 层级与结构
- **# 一级标题**：页面标题（可选，可覆盖 Frontmatter title）。
- **## 二级标题**：**核心单位（Card）**。每个 `##` 标题对应一个展示卡片。
- **### 三级标题**：**卡片子集（Set）**。在当前卡片内进一步划分。

### 3. 卡片元数据 (Card Metadata)
仅在 `##` 标题后紧邻的行内使用 `--- ... ---` 块来定义属性：
```md
## 🧭 模块名称
---
lang: bash
emoji: 🧭
link: https://example.com/docs
desc: 模块的一句话描述
---
```
- `link`：指向该模块的官方文档链接，必须是有效 URL。
- `desc`：模块说明节点。

### 4. 条目写法
- **命令项**：`- `命令` : 描述`。
- **普通项**：`- 描述文本`。
- **代码块**：渲染为 `<pre><code class="language-xxx">`。必须带语言标识。

## 🏷️ 元数据与参考规范

### meta.yml 结构
```yaml
desc: cheatsheet 的一句话描述
tags: ["标签1", "标签2"] # 参考分组：平台、工具类型、使用阶段、交互形态
version: "整理时参考的信息版本"
github: "owner/repo"
date: "YYYY-MM-DD"
```

### refmap.md 结构
按分类提供参考资源，确保每个核心模块皆有出处。

## 🚀 执行流程

1. **识别与创建**：识别主题，创建目录 `cheatsheets/<topic>/`。
2. **深度采集**：执行采集策略，整理出 8-15 个核心卡片内容。
3. **内容成稿**：
   - 编写 `<topic>.md`：严格执行语法规范与编排策略。
   - 编写 `meta.yml`：补齐所有必选元数据。
   - 编写 `refmap.md`：建立链接地图。
4. **自检验收**：执行改进策略中的自检规则，确认未生成 `.html`。

## 💡 提示

生成后提示用户：
`node cheatsheet-html-maker/index.js --input cheatsheets/<topic>/<topic>.md`
然后自动执行, 除非用户要求不执行
