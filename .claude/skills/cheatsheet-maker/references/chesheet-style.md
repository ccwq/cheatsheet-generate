# cheatsheet 默认风格指南

这份参考用于普通 `cheatsheet` 风格，不承担 cookbook 的任务引导职责。目标只有一个：让读者在最短时间内定位到可直接使用的命令、参数、语法和示例。

## 适用信号

- 主题更像命令表、参数表、语法表、配置表、API 速查
- 读者主要问题是“这一条怎么写”或“这个参数怎么用”
- 资料来源以 reference、manual、API docs、CLI help 为主
- 场景说明是辅助信息，检索效率才是主目标

## 结构目标

- 模块划分稳定，一眼能扫到
- 标题尽量直接对应能力或对象，不写虚词
- 每张卡片优先给命令、语法、参数、返回值、差异点
- 示例短、准、可复制
- 解释只保留足够理解该命令或片段的最小信息

## 推荐写法

- 先列高频命令，再列常用参数和变体
- 优先用表格、短列表、短代码块组织信息
- 同类项保持一致顺序，例如：用途 -> 命令 -> 说明 -> 注意点
- 一个代码块只展示一个主要意图，避免把多个知识点塞在一起

## 推荐章节

- 核心命令 / 常用语法
- 高频参数 / 常见选项
- 输入输出 / 返回值 / 结果差异
- 常见组合写法
- 易错点 / 注意事项

## 不要这样写

- 大段背景介绍
- 用 workflow 叙事把简单命令拖长
- 为了“完整”罗列低频边角参数
- 让读者翻多个章节才能找到一条命令

## 自检

1. 读者能否在 10 秒内找到想抄的命令或语法
2. 同类信息是否保持一致格式
3. 是否删掉了与使用无关的大段背景
4. 示例是否短小且能直接复制
