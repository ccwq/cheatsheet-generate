# Cheatsheet Review Rubric

## 适用范围

用于评审单个 `cheatsheets/<topic>/` 的质量，而不是跨 topic 排名。

## 评审维度

### 1. 内容完整度

检查该 topic 是否覆盖读者完成高频任务所需的最小知识集：

- 主题定位是否清楚
- 是否有核心命令 / API / 语法 / 配置项
- 是否有高频场景或最小工作流
- 是否有关键参数、分支决策、常见坑
- 是否遗漏决定性模块或子能力

### 2. 事实正确性

检查正文与元数据中是否存在可核实错误：

- 命令、参数、代码示例错误
- 概念解释错误
- 版本、仓库、发布日期错误
- 功能归因错误
- 引用来源和正文结论对不上

### 3. 版本时效性

检查内容是否明显过时：

- 主版本已变化但正文仍围绕旧版能力
- 命令或参数已废弃
- `meta.yml.version` 与实际主参考版本不一致
- `date` 与版本发布时间明显冲突

### 4. 结构适配度

检查内容组织是否适合主题：

- 单纯 reference 主题却写成空泛流程文
- 场景驱动主题却只罗列零散命令
- 混合型主题缺少入口卡、任务链或速查卡
- 章节顺序妨碍检索效率

### 5. 来源可追溯性

检查是否能从 `refmap.md` 和官方资料追到关键结论：

- 正文关键模块在 `refmap.md` 中找不到来源
- 链接失效或归类混乱
- 关键结论没有官方支撑

### 6. topic 内文件一致性

检查 `<topic>.md`、`meta.yml`、`refmap.md` 是否互相支持：

- `meta.yml.github`、`version`、`date` 与正文或官方事实冲突
- 正文大量提及的能力没有来源映射
- `refmap.md` 主要指向的对象和正文主题不一致

## 评级规则

### A 通过

满足以下特征：

- 没有发现高风险事实错误
- 核心内容覆盖较完整
- 结构与主题匹配
- 文件间一致性良好
- 即使有小问题，也不影响读者正确使用

### B 轻微问题

满足以下任一特征：

- 存在少量缺漏，但不影响主线使用
- 有个别表述、示例或元数据需要修正
- 可追溯性或结构存在轻微问题，但整体仍可发布

### C 明显匮乏或存在错误

满足以下任一特征：

- 缺少多个高频内容块，读者无法顺畅完成主要任务
- 出现会误导读者的事实错误，但影响仍局部
- 结构明显不适配主题，导致可用性差
- `meta.yml` 或 `refmap.md` 与正文存在明显脱节

### D 高风险不可用

满足以下任一特征：

- 多处关键命令、参数、概念或版本信息错误
- 主要内容明显过时，继续使用风险高
- 缺少核心入口与核心能力说明，基本不可用
- 文件一致性严重失真，无法判断正文依据

## 输出模板

```md
总评级：A | B | C | D

结论摘要：
- 用 2 到 4 句说明是否可发布，以及主要风险

主要问题：
- 按严重程度列出问题

内容匮乏项：
- 列出缺失的关键内容块

错误项：
- 列出事实错误、版本错误、命令错误、映射错误

证据与来源：
- 每条关键判断都给出对应官方来源或仓库依据

修复优先级建议：
- P0：必须先修
- P1：建议尽快补
- P2：可后续优化
```

## 判定提醒

- 不按字数机械打分
- 不因写得短就默认匮乏，也不因写得长就默认完整
- 只要涉及版本、命令、API、发布日期、仓库事实，就优先联网核验
- 若证据不足，标记“待核验”，不要伪造确定性
