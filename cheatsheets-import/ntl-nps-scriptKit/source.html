ntl (Node Task List)

背景介绍

ntl 是一个轻量级的终端工具，旨在让开发者更方便快捷地浏览并执行 package.json 中定义的 npm 脚本。它强调交互式操作和脚本快速调用，是管理和执行复杂 npm 脚本任务的利器。

安装与启动

使用全局命令安装：

npm install -g ntl

进入含有 package.json 的项目根目录，执行启动命令：

ntl

核心功能与操作

  • 显示 package.json 中所有 npm 脚本的交互式列表，可视化任务查看
  • 使用上下键选择脚本，回车执行脚本快速调用
  • 支持模糊匹配搜索脚本名称，提高定位效率
  • Tab 键切换列表视图与搜索视图，操作灵活
  • 支持快捷键执行最近一次运行的脚本，快速重复操作
  • 脚本运行的输出直接在当前终端显示，便于调试
  • 通过配置环境变量或 package.json，可以自定义使用的包管理器（npm、yarn、pnpm 等）

脚本描述和详细信息

在 package.json 中添加 ntl.descriptions 字段，可以为每个脚本添加描述信息，有助于团队理解任务含义：

{
  "scripts": {
    "build": "make build",
    "test": "jest"
  },
  "ntl": {
    "descriptions": {
      "build": "编译项目",
      "test": "执行测试"
    }
  }
}

使用 --info 或 -i 参数开启任务详细信息显示：

ntl -i  # 或 ntl --info

运行多个任务

支持多选任务，允许按序列顺序执行多个脚本，提升执行效率：

ntl -m  # 或 ntl --multiple

通过空格键选择多个任务，回车执行。结合 --ordered 参数可按照选择顺序依次执行。

快捷运行与项目集成

  • 输入命令 nt 快速执行上一次运行的脚本，节省操作步骤
  • 可在项目的 package.json 中添加 "start": "ntl" 脚本，为团队成员提供统一的脚本管理入口
  • 支持配置忽略指定脚本，使界面更简洁，聚焦常用任务

nps (Node Package Scripts)

背景介绍

nps 是一个基于 JavaScript 配置的 npm 脚本管理工具，采用 package-scripts.js 统一管理复杂和嵌套的脚本。相比传统 package.json，nps 强调脚本复用、可组合和跨平台环境友好，适用于大型项目和团队合作。

安装与初始化

安装 nps 及相关工具库：

npm install --save-dev nps nps-utils

初始化生成配置文件：

npx nps init

配置文件结构与示例

在项目根目录生成 package-scripts.js 文件，通过 JavaScript 方式灵活管理脚本：

const { series, rimraf, crossEnv } = require('nps-utils')

module.exports = {
  scripts: {
    clean: rimraf('dist'),                  // 清理 dist 目录
    build: series.nps('clean', 'webpack'), // 按顺序执行 clean 和 webpack
    start: crossEnv('NODE_ENV=production', 'node server.js'), // 设置环境变量启动服务
    test: 'jest --coverage',                // 运行测试并生成覆盖率报告
    lint: {
      js: "eslint .",                       // 嵌套脚本，可点访问
      css: "stylelint '**/*.css'"
    }
  }
}

package.json 中调用

通过 nps 组织完善脚本后，在 package.json 使用简洁的引用调用：

{
  "scripts": {
    "build": "nps build",
    "start": "nps start",
    "test": "nps test",
    "lint:js": "nps lint.js"
  }
}

优势与常用技巧

  • 跨平台环境变量设置的安全与标准化
  • 支持脚本嵌套与点语法访问，方便管理复杂脚本
  • 可组合脚本实现串行或并行，提升构建效率
  • 适用于大型项目和团队脚本规范化管理
  • 丰富的工具函数库（nps-utils）提高脚本复用率

Script Kit

背景介绍

Script Kit 由 John Lindquist 设计，是一个基于 TypeScript/JavaScript 的桌面自动化脚本平台，提供交互式脚本运行环境。它集成了丰富的系统支持和提示功能，适合快速开发 UI 脚本、文件管理、系统集成等自动化工具。

安装与启动

全局安装并启动交互式脚本界面：

npm install -g @johnlindquist/kit
kit

脚本文件结构和编辑

脚本采用 TypeScript/JavaScript 编写，推荐存放在默认目录 ~/.kit。脚本文件顶部引用官方 SDK 来访问内置 API：

import "@johnlindquist/kit";

常用 API 与示例

Script Kit 提供丰富实用帮助函数，快速开发交互式脚本：

import "@johnlindquist/kit";

let name = await arg("请输入名字");
await showToast(`你好，${name}!`);

let files = await getFiles("*.js");
console.log(files);

API 说明

  • await arg(prompt)：弹出输入框，获取用户输入
  • await showToast(message)：显示桌面通知
  • env：访问和操作环境变量
  • exec(command)：执行 Shell 命令并返回结果
  • writeFile(path, content) / readFile(path)：文件读写操作
  • getFiles(pattern)：根据通配符匹配查找文件

特点与适用场景

  • 零配置启用，自动类型提示辅助开发
  • 适合打造桌面自动化的小工具及交互式脚本
  • 丰富的系统交互能力，包括剪贴板、通知、窗口操作
  • 个人及团队自动化脚本开发的高效平台
