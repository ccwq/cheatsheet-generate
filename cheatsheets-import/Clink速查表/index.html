Clink 概述与安装

Clink 将 Windows cmd.exe 与强大的 GNU Readline 命令行编辑功能（如 Bash 中所知）相结合，提供增强的输入、历史和补全功能。

核心亮点

  • Auto-Suggestions: 基于历史和文件实时建议。
  • Completions: Tab/Ctrl-Space 补全可执行文件、别名、命令参数。
  • Persistent History: 跨会话保存命令历史。
  • Scriptable Prompt: 通过 Lua 脚本定制动态彩色提示符。
  • 便捷操作: 目录快捷方式、增强 Doskey 扩展。

安装方式

# 使用 winget 安装
winget install clink
# 或使用 scoop
scoop install clink
# 或运行 Setup EXE/下载 ZIP.

启动 Clink (注入)

  • Autorun: 安装时勾选自动运行，或运行 clink autorun install。
  • 手动启动: 运行安装目录下的 clink.bat 或快捷方式。
  • 注入现有进程: 在未加载 Clink 的 cmd 窗口中运行 clink inject。

核心功能快捷键

Clink 默认采用 Bash 键绑定 (Ctrl-A/E/F 等)，但可配置为 Windows 默认。

常用编辑命令

Alt-H # 显示所有活动键绑定
Tab # 完成/循环补全 (取决于配置)
Ctrl-Space # 交互式补全列表 (clink-select-complete)
Ctrl-Z # 撤销 (undo)
Ctrl-V # 粘贴 (clink-paste)
Shift-Arrows # 选择文本 (CUA mode)

命令历史管理

  • Up / Down: 循环历史条目。
  • PgUp / PgDn: 循环匹配已输入前缀的历史条目。
  • Ctrl-R / Ctrl-S: 增量搜索历史 (Ctrl-R: 向后搜索)。
  • Ctrl-Alt-Up: 弹出历史列表 (clink-popup-history)。
  • Alt-. / Alt-_: 插入上一条命令的最后一个参数 (yank-last-arg)。
  • Alt-Ctrl-K: 添加当前行到历史但不执行 (add-history)。
历史记录默认保存在 clink_history 文件中，上限 10000 行。

补全与自动建议

"补全" 针对光标处的单词；"建议" 针对整条命令，通常基于历史。

自动建议 (Auto-Suggest)

当输入匹配历史命令时，会以静音颜色显示建议。
  • Right / End: 插入整个建议。
  • Ctrl-Right: 插入建议的下一个单词。
  • F2: 切换显示交互式建议列表。

补全行为配置

# 启用/禁用自动建议
clink set autosuggest.enable false
# 建议策略 (优先级从高到低)
clink set autosuggest.strategy history completion
# 切换 Windows/Bash 键绑定风格
clink set clink.default_bindings windows
# 启用环境变量展开匹配
clink set match.expand_envvars true

完成命令 (Completion Commands)

  • complete: 尝试完成光标前文本 (Tab 默认行为)。
  • old-menu-complete: 循环替换可能的补全项 (Windows/增强默认 Tab 行为)。
  • clink-select-complete: 交互式列表，支持过滤 (Ctrl-Space 默认行为)。

Clink 基础设置 (clink set)

使用 clink set 命令管理 Clink 运行时设置。

常见配置

# 显示所有设置和值
clink set
# 设置历史记录最大行数 (默认 10000)
clink set history.max_lines 25000
# 忽略以空格开头的命令 (不保存到历史)
clink set history.ignore_space true
# 自动应答 "Terminate batch job?" 提示
clink set cmd.auto_answer answer_yes
# Ctrl-D 是否退出 CMD (默认 true)
clink set cmd.ctrld_exits false
# 粘贴时如何处理 CR/LF (crlf/delete/space/ampersand)
clink set clink.paste_crlf space

输入与着色

  • clink.colorize_input: 启用上下文敏感输入文本着色 (默认 true)。
  • autosuggest.original_case: 插入建议时保留原始大小写 (默认 true)。
  • doskey.enhanced: 增强 Doskey 扩展 (支持 & 和 | 之后，尊重引号)。

Readline 配置 (.inputrc)

用于配置 Readline 库变量和键绑定。通常位于 %USERPROFILE%\.inputrc。

.inputrc 文件结构

# 注释行
set colored-stats on # 设置变量
C-a: beginning-of-line # 绑定命令
"\e[2~": "luafunc:my_func" # 绑定宏

$if clink # Clink 特有条件块
set search-ignore-case on
$endif

常用 Readline 变量

  • editing-mode: emacs (默认) 或 vi。
  • colored-stats: on 启用补全项着色（需配合 match.coloring_rules）。
  • completion-ignore-case: on 文件名匹配忽略大小写。
  • history-preserve-point: on 历史导航时保留光标位置。
  • mark-directories: on 目录补全时添加斜杠。

剪切/粘贴 (Kill & Yank)

Kill 将文本存入 kill-ring（剪切环），Yank 粘贴。不影响系统剪贴板。

Killing (剪切)

  • Ctrl-End: 剪切从光标到行尾的文本。
  • Ctrl-Del: 剪切光标到词尾的文本。
  • Ctrl-Backspace: 剪切光标到词首的文本。
  • Ctrl-W: 剪切光标到前一个空格边界的文本。

Yanking (粘贴)

  • Ctrl-Y: 粘贴最近剪切的文本。
  • Alt-Y: 循环剪切环，粘贴前一个剪切的文本 (只能紧跟在 Ctrl-Y 或 Alt-Y 之后使用)。
  • Ctrl-V: 粘贴系统剪贴板内容 (clink-paste)。

目录快捷方式

用于快速切换目录，仅在交互式命令行中生效。
  • dirname\: 相当于 cd /d dirname。
  • .. / ...: 相当于 cd .. / cd ..\..。
  • - / cd -: 切换到上一个工作目录。

颜色配置与主题

使用 clink set color.NAME VALUE 或 .clinktheme 文件。

颜色值语法 (SGR 参数)

# [属性] [前景色] [on [背景色]]
bold underline bright white on blue
# RGB Hex 格式 (v1.7.0+)
bold green on #222222
# 设置参数颜色
clink set color.arg "bold yellow"
# 设置自动建议颜色 (默认 dim/bright black)
clink set color.suggestion "bright black"

主题管理

  • clink config theme list: 列出可用主题。
  • clink config theme use ThemeName: 应用主题。
  • clink config theme save MyTheme: 保存当前颜色设置为新主题。

输入文本着色分类

  • color.argmatcher: 带有 Argmatcher 的命令名。
  • color.cmd: CMD 内置命令。
  • color.doskey: Doskey 别名。
  • color.flag / color.arg: 命令的旗帜/参数。
  • color.unexpected: 无法识别的参数。

自定义提示符

通过 Lua 脚本 (`clink.promptfilter`) 或 .clinkprompt 文件高度定制提示符样式。

提示符过滤 (Lua)

local p = clink.promptfilter(30) function p:filter(prompt) local git_branch = git.getbranch() if git_branch then prompt = "[" .. git_branch .. "] " .. prompt end return prompt end
  • 优先级 (Priority): 低数字先执行。
  • clink.promptfilter(P):rightfilter(): 定制右侧提示符 (RPrompt)。
  • clink.promptfilter(P):transientfilter(): 定制瞬态提示符 (用于历史命令)。
  • 异步提示: 使用 clink.promptcoroutine(func) 在后台运行慢速命令。

Prompt 配置

  • clink config prompt list: 列出可用提示符模块。
  • clink config prompt use PromptName: 应用自定义提示符。
  • prompt.transient: 设置瞬态提示符模式 (off, always, same_dir)。

Lua 扩展 API (核心)

Clink 通过 Lua 5.2 扩展功能，例如定义自定义补全逻辑。

Argument Matcher (_argmatcher)

定义命令参数的结构和补全项。
local parser = clink.argmatcher("git") parser:addarg({ "merge", "pull" }) :addflags("--help", "--verbose") :adddescriptions({ "--help", description="Show help text" })
  • :addarg(): 添加位置参数（可传入 clink.filematches/ clink.dirmatches）。
  • 链接解析器: 将参数链接到另一个解析器（用于子命令）。
  • :chaincommand(): 使后续文本被解析为新命令。
  • Adaptive: 使用 :setdelayinit() 实现延迟初始化。

实用 Lua APIs

# 缓冲区操作 (rl_buffer)
rl_buffer:insert("text")
rl_buffer:getbuffer()
# 调用命令
rl.invokecommand("yank")
# 获取 Git 状态 (需要 Git)
git.getbranch()
git.getstatus()

杂项与高级操作

历史扩张 (History Expansion)

类似于 Csh 的历史操作，默认启用 (`!`)。
  • !!: 重复上一条命令。
  • !n: 第 n 条历史命令。
  • !-n: 倒数第 n 条历史命令。
  • !string: 最近以 string 开头的命令。
  • ^str1^str2^: 快速替换上一条命令中的 str1 为 str2。

鼠标与滚动

  • clink set terminal.mouse_input auto: 启用鼠标输入（点击设置光标）。
  • Alt-PgUp / Alt-PgDn: 滚动控制台窗口。
  • Shift-Left/Right/Home/End: CUA 模式下选择文本。

调试与信息

  • clink info: 打印版本、配置目录和文件信息。
  • clink echo: 打印按键对应的原始键序列，用于配置 .inputrc。
  • clink update: 检查并安装更新。

可绑定命令 (部分)

可使用 keyname: command 绑定到 .inputrc。

移动与改变文本

  • beginning-of-line: 移动到行首。
  • end-of-line: 移动到行尾。
  • transpose-chars: 交换光标前后字符。
  • upcase-word: 单词转大写 (Alt-U)。
  • downcase-word: 单词转小写 (Alt-L)。

Clink 扩展命令

  • clink-copy-line (Alt-Ctrl-C): 复制当前行到系统剪贴板。
  • clink-copy-word (Alt-Ctrl-W): 复制当前词到系统剪贴板。
  • clink-expand-line (Alt-Ctrl-E): 展开历史、别名和环境变量。
  • clink-toggle-slashes (Ctrl-/): 切换光标处单词中的斜杠方向。
  • remove-history (Alt-Ctrl-D): 在历史搜索中删除当前行。
  • edit-and-execute-command (Ctrl-X Ctrl-E): 使用编辑器编辑当前行并执行。
