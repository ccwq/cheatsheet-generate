断点类型与操作

断点是调试的基础，熟练掌握不同类型的断点可以极大地提高效率。

普通断点

在代码行号旁点击即可设置，程序运行至此会暂停。

条件断点

右键点击断点，输入布尔表达式。当表达式为真时，断点才会触发。

for (int i = 0; i < items.size(); i++) {
    // 当 i == 5 时触发断点
    // condition: i == 5
}

日志断点 (Non-suspending Breakpoints)

右键断点 -> `More...`，取消勾选 `Suspend`，并在 `Log` 选项中打印变量值或自定义信息。它不会中断程序执行。

for (int i = 0; i < items.size(); i++) {
    // log message: "当前 i 的值为: " + i
}

异常断点

在调试窗口的断点(Breakpoints)面板中，点击`+`号选择`Java Exception Breakpoints`，可设置当抛出特定异常时暂停。

调试面板核心功能

了解调试面板的各个区域是高效调试的关键。

变量 (Variables)

  • 实时查看当前作用域中的所有变量值。
  • 右键变量可以`Watch`（监视）或`Set Value`（修改值），甚至在运行时改变变量状态。
  • 对于复杂的对象，可以展开并查看其内部成员变量。

帧 (Frames)

  • 显示当前的调用栈，从上到下为最近到最远的调用。
  • 点击任一帧可以查看该函数调用时的变量状态和执行点。

控制台 (Console)

  • 显示应用程序的日志输出（Logcat）。
  • 可以执行`Log.d()`等打印语句。

实用调试操作

一些快捷操作可以帮助你快速定位问题。

步进与步出

  • **Step Over (F8)**: 单步执行，跳过函数内部。
  • **Step Into (F7)**: 进入函数内部进行调试。
  • **Force Step Into (Alt+Shift+F7)**: 强制进入系统源码或第三方库内部。
  • **Step Out (Shift+F8)**: 从当前函数中跳出。

表达式求值 (Evaluate Expression)

在调试时，选择代码片段或在表达式求值器中输入表达式，可以实时查看其结果，常用于验证逻辑。

// 实时计算变量值或调用方法
// 表达式: new Date().getTime()
// 表达式: myObject.someMethod()
// 表达式: (myVariable + 10) / 2

实时编辑 (Live Edit)

Android Studio 提供了强大的实时编辑功能，允许你在不重新部署应用的情况下修改代码。

支持范围

  • 目前主要支持 Jetpack Compose，允许修改 composable 函数中的代码。
  • UI 会在设备或模拟器上实时更新，大幅提升开发效率。

快捷键

  • 默认情况下，修改后会自动应用。
  • 手动应用修改：`Ctrl + Alt + Shift + F9`。

Logcat 日志过滤

Logcat 是 Android 的标准日志工具，学会过滤可以快速找到关键信息。

过滤方式

  • **按级别过滤**: Verbose, Debug, Info, Warn, Error, Assert。
  • **按标签过滤**: `tag:MyTag`。
  • **按包名过滤**: `package:com.example.myapp`。
  • **按关键词过滤**: `text:MyKeyword`。

组合过滤

可以使用空格组合多个过滤条件，例如：

tag:Network AND text:Error

性能分析器 (Profilers)

Profiler 工具套件用于分析应用的性能，包括 CPU、内存、网络和电量。

内存分析器 (Memory Profiler)

  • 用于检测内存泄漏和内存抖动。
  • 可以强制垃圾回收（`Force garbage collection`）并捕获堆转储（`Capture heap dump`）。

CPU 分析器 (CPU Profiler)

  • 用于追踪方法调用，分析函数耗时。
  • 可以记录跟踪文件（`System Trace` 或 `Method Trace`）。

网络分析器 (Network Inspector)

在 Profilers 中，Network Inspector 专门用于监控应用的HTTP/HTTPS网络请求。

主要功能

  • **网络流量**: 实时查看应用的网络流量和数据传输量。
  • **请求详情**: 检查每个请求的 URL、请求方法、状态码和时间。
  • **请求/响应头和正文**: 详细查看请求头、响应头以及原始的请求体和响应体。

这对于调试 API 调用、验证数据格式以及诊断连接问题非常有帮助。

附加调试器到进程

如果你的应用已经在设备上运行，但不是通过 Android Studio 启动的，可以使用此功能进行调试。

应用场景

  • 调试系统应用或预装应用。
  • 调试通过其他方式（如命令行）启动的应用。

操作步骤

菜单栏: `Run` -> `Attach Debugger to Android Process`，然后选择需要调试的进程。

布局检查器 (Layout Inspector)

可视化地调试布局，查看视图的层次结构和属性。

主要用途

  • **视图层次结构**: 检查所有 View 或 Composable 的嵌套关系。
  • **属性检查**: 实时查看和修改视图的布局参数、颜色、文本等属性。

打开方式

运行应用后，在`Run`菜单下找到`Layout Inspector`，或点击工具栏上的相应图标。

原生代码调试 (NDK Debugging)

如果你的应用包含 C/C++ 代码，可以使用 Android Studio 进行联合调试。

配置步骤

在你的`build.gradle`文件中确保 `debuggable` 选项为 `true`。

android {
    buildTypes {
        debug {
            debuggable true
        }
    }
}

在需要调试的 C/C++ 代码行号旁设置断点即可。
