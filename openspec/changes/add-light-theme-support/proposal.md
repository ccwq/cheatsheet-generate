## Why

当前系统仅提供暗色主题，无法覆盖偏好亮色界面的用户场景，也限制了产品在白天办公、演示展示与品牌表达上的视觉适配能力。现在补齐亮色主题与主题切换能力，可以在不牺牲现有暗色体验的前提下，提升界面的一致性、可访问性与整体质感。

## What Changes

- 新增系统级明暗主题切换能力，支持在亮色与暗色之间切换，并保证切换后的界面状态一致。
- 为现有暗色设计补齐对应的亮色视觉规范，包括背景、文字、边框、分割线、卡片、悬浮层与交互状态。
- 统一整理主题令牌与组件样式映射，使页面在两套主题下都保持精美、优雅、高端的视觉表现。
- 预留主题持久化与系统偏好跟随能力，避免用户每次进入系统都需要重复选择。

## Capabilities

### New Capabilities
- `theme-mode-switching`: 提供应用级主题模式切换、默认主题判定、用户偏好保存与读取能力。
- `light-theme-visual-system`: 定义并应用亮色主题下的色板、文字层级、容器层级、状态色与交互反馈规范。

### Modified Capabilities
- 无

## Impact

- 影响前端样式体系，包括全局设计令牌、颜色变量、主题上下文与基础布局样式。
- 影响核心 UI 组件在不同主题下的展示方式，例如导航、卡片、表单、按钮、弹层与提示信息。
- 可能影响已有图标、插画、图片资源在亮色背景下的对比度与可读性，需要逐步校验。
- 不涉及外部 API 变更，但会影响前端渲染逻辑、主题存储策略与后续设计规范维护方式。
