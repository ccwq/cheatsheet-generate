# prompt-image-gen

## 一眼入口
---
emoji: 🍌
link: https://developers.googleblog.com/how-to-prompt-gemini-2-5-flash-image-generation-for-the-best-results/
desc: 这不是模型介绍，而是一份面向生图 prompt 的中文拼装词典。核心目标不是“堆形容词”，而是让你知道画面缺哪一层就补哪一层。
---
- 适合两类用法：
  - 从零拼一个完整 prompt
  - 拿已有 prompt 查漏补缺
- 最稳定的起手顺序：
  `主体 -> 场景 -> 构图 -> 镜头 -> 光线 -> 色彩 -> 材质 -> 氛围 -> 动作 -> 文字 -> 负面限制`
- 不知道先看哪：
  - 画面太空：先看 `场景 / 前中后景 / 环境元素`
  - 画面太平：先看 `构图 / 镜头 / 光线`
  - 画面太假：先看 `材质与表面痕迹`
  - 画面没情绪：先看 `色彩 / 氛围 / 天气`
  - 海报字不好：先看 `文字排版 / 标题控制`

## 怎么用这份词典
---
emoji: 🧭
link: assets/nano-banana-guide/index.html
desc: 读法按“先搭骨架，再补视觉层，最后收口”来理解。可以把它类比成前端页面：先写结构，再补样式，最后修边界条件。
---
- 第 1 层，先定骨架：
  `主体 + 场景 + 构图`
- 第 2 层，再补视觉：
  `镜头 + 光线 + 色彩 + 材质 + 氛围`
- 第 3 层，最后收口：
  `动作 / 姿态 + 文字 + 负面限制`
- 本页结构建议这样扫：
  - 先看 `最小拼装模板`
  - 再按问题看 `基础维度词典`
  - 最后去 `专题模板` 直接抄组合

## 最小拼装模板
---
emoji: 🧩
link: https://developers.googleblog.com/how-to-prompt-gemini-2-5-flash-image-generation-for-the-best-results/
desc: 从零写 prompt 时，先套模板再补词，通常比直接想到什么写什么更稳。
---

```text
[主体]
in / at [场景]
[构图与主体位置]
[镜头 / 景别 / 机位 / 景深]
[光线]
[色彩]
[材质与细节]
[氛围]
[动作 / 姿态 / 视线]
[文字排版要求，可选]
[负面限制]
```

- 极简版：

```text
年轻女性，雨夜街头，半身偏左构图，侧光与霓虹反射，冷蓝低饱和，湿发与皮夹克有细微反光，安静疏离，回头看向镜头，不要多余人物，不要手部变形
```

- 海报版：

```text
未来感耳机产品，悬浮展示，居中海报式构图，冷色轮廓光，金属与玻璃折射清晰，深色背景留白，顶部大标题清晰可读，右下角小标签，不要文字变形，不要遮挡主体
```

## 画面出问题时先补哪一层
---
emoji: 🛠️
link: assets/nano-banana-guide/index.html
desc: 多数失败图不是“模型不行”，而是 prompt 在某一层描述严重缺失。先定位问题，再补最相关的层，不要一口气全改。
---
- 只有一个主体名词，没有空间感：
  补 `场景 + 前中后景 + 地面 / 背景元素`
- 有主体有背景，但像证件照：
  补 `构图 + 景别 + 机位 + 景深`
- 词很多，但画面没成立：
  补 `主光方向 + 光质 + 阴影关系`
- 画面干净但像塑料：
  补 `材质 + 反光方式 + 表面痕迹`
- 图是对的，但没情绪：
  补 `色彩倾向 + 饱和度 + 天气 / 空气`
- 动作存在，但仍像摆拍：
  补 `姿态 + 重心 + 手部任务 + 视线`
- 海报里有字，但像随机贴上去：
  补 `位置 + 层级 + 留白 + 可读性`

## 基础维度词典：主体系统
---

desc: 主体系统回答“谁 / 什么是视觉中心”。这里优先补身份、状态和可见特征，而不是只堆抽象审美词。

- 金属装置、机械臂、复古相机、玻璃瓶、香水瓶、耳机、手表、跑车、悬浮设备、未来终端、甜点、咖啡杯、建筑模型

### 外形与气质
- 清冷感、松弛感、锋利感、脆弱感、疲惫感、精致感、野性气质、书卷气、未来感、街头感、贵气、纪实感

### 脸部与发型
- 短发、长卷发、湿发、凌乱发丝、高马尾、空气刘海、利落短发、轮廓分明、五官柔和、眼神锐利、目光空洞、轻微雀斑

### 服装与配饰
- 机能夹克、长风衣、皮夹克、廓形西装、丝质长裙、针织毛衣、工装裤、银色耳机、细框眼镜、金属项链、皮质手套

### 组合提醒
- 最稳定的骨架：
  `主体身份 + 外形气质 + 服装配饰`
- 如果主体总是漂：
  先少写抽象气质，多写可见外形和穿着

  `姿态母类 + 重心细节 + 手部任务 + 视线方向`
- 如果人物很僵：
  优先补 `重心` 和 `手部任务`

## 最常见错误
---
emoji: ⚠️
link: assets/nano-banana-guide/index.html
desc: 这部分只保留最容易导致画面失控的错误，方便写 prompt 前快速自检。
---
- 一句话塞太多细节，主次不分
- 同时改太多变量，导致每轮都漂
- 风格锚点过多，彼此冲突
- 只写抽象词，不写可见元素
- 只写“高级感”“电影感”，不写镜头和光线
- 材质名写了，但没有反光、痕迹和触感
- 背景只写“很酷”，没有地点、天气和地面状态
- 负面限制过长，反而稀释重点
