⚙️ CLI高级选项 >>>

基础选项

  • --help / -h : 显示帮助信息
  • --version / -v : 显示详细版本信息
  • --debug [filter] : 启用调试模式，支持过滤(如"api,hooks"或"!statsig")
  • --verbose : 覆盖详细模式设置

交互模式选项

  • -p, --print : 打印响应后退出(管道模式)
  • -c, --continue : 继续最近的对话
  • -r, --resume [sessionId] : 恢复指定ID的对话
  • --fork-session : 恢复时创建新会话ID

配置和设置

  • --settings : 从JSON文件或字符串加载额外设置
  • --model : 指定AI模型(如'sonnet'或完整模型名)
  • --fallback-model : 设置默认模型超载时的备用模型
  • --system-prompt : 指定会话系统提示
  • --append-system-prompt : 追加系统提示到默认提示

权限和安全

  • --permission-mode : 权限模式(acceptEdits/bypassPermissions/default/plan)
  • --dangerously-skip-permissions : 跳过所有权限检查
  • --allowed-tools : 指定允许的工具列表
  • --disallowed-tools : 指定禁止的工具列表

MCP和插件

  • --mcp-config : 从JSON文件/字符串加载MCP服务器
  • --plugin-dir : 加载插件目录(本次会话)
  • --agents : JSON格式定义自定义代理

会话控制

  • --session-id : 使用指定UUID进行对话
  • --add-dir : 允许工具访问的额外目录
  • --output-format : 输出格式(text/json/stream-json，需配合--print)
所有选项基于Claude Code 2.0.37版本验证

  • claude mcp get 名称 : 获取MCP服务器详情
  • claude mcp add-json 名称 JSON : JSON方式添加服务器

添加方式示例

# HTTP服务器
claude mcp add --transport http sentry https://mcp.sentry.dev/mcp

# SSE服务器
claude mcp add --transport sse asana https://mcp.asana.com/sse

# stdio服务器
claude mcp add --transport stdio airtable --env AIRTABLE_API_KEY=KEY -- npx -y airtable-mcp-server

🔐 权限管理 >>>

基础权限命令

  • /permissions show : 显示当前权限
  • /permissions reset : 重置权限设置

  • /permissions shell-allowed : 允许Shell访问
  • /permissions sandbox-only : 仅沙盒模式

  • /permissions network-offline : 离线模式

  • /permissions mcp-logs : 显示MCP访问日志
