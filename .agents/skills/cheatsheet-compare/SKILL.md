# cheatsheet-compare

对 `cheatsheets/<topic-a>/` 和 `cheatsheets/<topic-b>/` 进行多维度结构化对比，输出可辅助决策的对比报告。

## 核心产物

结构化对比报告，包含：
- 基本信息对比
- 定位差异分析
- 功能矩阵对比
- 优劣势对比
- 适用场景推荐
- 综合评价与二选一建议

## 兼容性

- 依赖文件读取能力
- 依赖 `assets/report-template.md` 模板（如需参考格式）
- 必要时可调用 `web-search` 核对版本信息

## 触发条件

满足任一场景时触发：
- 用户明确要求"对比"、"比较"两个 cheatsheet topic
- 用户询问"xxx 和 xxx 哪个更好"、"二选一推荐"
- 用户要求分析两个工具的"差异"、"区别"
- 用户要求做"优劣对比"、"全方位对比"

**不触发本技能的场景**：
- 用户只要求评审单个 cheatsheet 的质量（转交 `cheatsheet-review-skill`）
- 用户要求更新单个 cheatsheet（转交 `cheatsheet-update`）
- 用户要求创建新 cheatsheet（转交 `cheatsheet-maker`）
- 用户要求补标签、补图标等单项维护任务（转交对应技能）

## 输入参数

必须明确两个目标 topic：
- `topic-a`：第一个要比较的主题目录名（不含路径）
- `topic-b`：第二个要比较的主题目录名（不含路径）

用户可以用以下方式表达：
- "对比 bash 和 zsh"
- "比较 x-cmd 和 oh-my-zsh 的优劣"
- "npm vs yarn 选哪个好"

## 执行流程

### 1. 解析 topic 名称

从用户输入中提取两个 topic 名称，然后在 `cheatsheets/` 目录下匹配。

### 2. 读取目标 cheatsheet

对每个 topic 分别读取：
- `cheatsheets/<topic>/<topic>.md`（主内容）
- `cheatsheets/<topic>/meta.yml`（元数据）
- `cheatsheets/<topic>/refmap.md`（参考来源）

若任一 topic 不存在：
- 若另一个 topic 存在，询问用户是否要对比已存在的 topic 与另一个不存在的 topic
- 若两个都不存在，返回错误并列出可用 topic 供用户选择

### 3. 提取对比维度（7-8 个通用维度）

| 维度 | 说明 |
|------|------|
| 定位 | 这是什么工具，解决什么问题，目标用户是谁 |
| 核心能力 | 主要功能模块、命令集、API 能力 |
| 安装方式 | 支持的平台，安装难度 |
| 配置复杂度 | 初始配置难度，定制化程度 |
| 性能特征 | 资源占用、启动速度、执行效率 |
| 扩展生态 | 插件系统、主题、模块市场 |
| 学习曲线 | 上手难度，文档质量 |
| 维护状态 | 版本活跃度，更新频率 |

### 4. 定位差异分析

- 读取两个 topic 的 `meta.yml` 的 `desc` 字段
- 读取两个 topic 正文的"定位"相关段落
- 判断两个工具是否解决同一类问题
- 判断两个工具是否可以直接替代，还是解决不同层面的问题

### 5. 功能矩阵对比

按以下分类逐项对比：
- **通用能力**：安装平台、依赖要求、配置文件格式
- **日常高频**：日常使用的核心命令、交互方式
- **扩展生态**：插件数量、主题数量、社区活跃度
- **自动化支持**：脚本化能力、API 接口、CI/CD 集成
- **运维能力**：调试功能、日志输出、错误处理

### 6. 优劣势提炼

基于两个 cheatsheet 的内容，分别提炼：

**优势项**（每个 topic 不超过 5 条）：
- 该工具明显优于对方的地方
- 必须是正文或官方资料中有依据的

**劣势项**（每个 topic 不超过 5 条）：
- 该工具明显弱于对方的地方
- 必须是正文或官方资料中有依据的

### 7. 适用场景推荐

给出 3-6 个典型场景，判断每个场景下哪个工具更合适：

| 场景 | 推荐 | 理由 |
|------|------|------|
| 场景描述 | topic-a / topic-b / 两者皆可 | 简要说明 |

### 8. 综合评价

给出 2-3 句总结性评价，说明：
- 两个工具的核心差异是什么
- 各自最适合什么样的用户群体
- 不存在绝对优劣，只有场景适配

### 9. 输出报告文件

将对比报告写入文件：`cheatsheet-compare/<任务名称>.md`

- `任务名称` 由两个 topic 名称拼接而成，格式为 `<topic-a>-vs-<topic-b>`
- 例如：对比 agent-browser 和 playwright-cli 时，文件名为 `agent-browser-vs-playwright-cli.md`
- 目录 `cheatsheet-compare/` 位于项目根目录下，与 `cheatsheets/` 同级
- 报告内容采用上述固定结构

## 自检清单

- 两个 topic 都存在于 `cheatsheets/` 目录下
- 对比维度覆盖定位、功能、使用体验、生态（7-8 个维度）
- 优势和劣势各有依据，来源于正文或 refmap
- 适用场景不少于 3 个
- 输出格式完整，不缺少任何固定栏目
- 使用简体中文输出
- 报告文件已保存至 `cheatsheet-compare/<topic-a>-vs-<topic-b>.md`

## 协作边界

- 本技能只做对比分析，不修改任何 cheatsheet 内容
- 本技能不做单个 topic 的质量评审
- 如用户要求对比的 topic 不存在，优先使用现有 cheatsheet，无则转交 `cheatsheet-maker` 创建
