# cookbook + cheatsheet 混合风格指南

这份参考用于“既要速查，又要场景带路”的主题。它不是把两篇文档强行拼接，而是让 `cookbook` 负责组织任务路径，让 `cheatsheet` 负责压缩关键动作。

## 适用信号

- 主题同时存在高频命令和高频场景
- 读者既会搜索单条命令，也会搜索“某种情况该怎么做”
- 来源既有 reference，也有 guides / recipes / workflows
- 单纯做成 cheatsheet 会缺少上下文，单纯做成 cookbook 又会降低检索效率

## 组织原则
- 前面的卡片按照cookbook, 之后使用cheatsheet再次整理内容
- 开头先给最小脑图：这是什么、先看哪、最短路径是什么
- 主体按场景或 workflow 拆段
- 每段内部嵌入速查块，而不是只写叙述
- 速查块优先回答“要敲什么、有什么分支、默认行为是什么”
- 章节之间尽量形成“先做什么，再看什么”的路径感

## 推荐章节

- 快速定位
- 最小工作流
- 高频场景 recipes
- Quick Ref / 命令速查
- 常见分支决策
- 风险点 / 排障 / 收尾

## 适合嵌入的速查块

- `标签 + 命令 + 解释`
- 参数对照表
- 常见分支的 A / B 选择
- 场景到命令链的映射
- 模块职责速记

## 常见失败写法

- 只有 workflow，没有可扫读的关键命令
- 只有命令卡，没有告诉读者什么时候用哪组命令
- 同一章节同时讲太多分支，导致卡片失去检索性

## 自检

1. 读者能否先按场景找到入口
2. 找到入口后，能否直接抄到关键命令
3. 有没有把同一信息重复写成两遍
4. 每个章节是否同时具备“带路”和“速查”中的至少一个强特征
