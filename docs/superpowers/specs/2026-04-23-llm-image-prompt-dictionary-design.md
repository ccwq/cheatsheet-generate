# 大语言模型生图提示词词典设计

## 背景

现有 `cheatsheets/prompt-image-gen/prompt-image-gen.md` 偏模型介绍、工作流和产品信息，不符合用户对“提示词查询手册”的预期。用户明确要求这份内容改为：

- 以中文为主
- 供用户从不同维度查询和补全画面描述
- 重点是“候选词语 / 短语片段”，而不是模型科普
- 仍然参考 `C:/Users/Administrator/ag-research/nano-banana-guide/index.html`
- 吸收参考页中高价值的微细节和画面控制表达，例如：
  - 尘埃颗粒
  - 微小划痕
  - 胶片颗粒
  - 柔和光晕
  - 细微高光
  - 环境雾气
  - 冰冷反射
  - 指纹痕迹

## 目标

把这份 cheatsheet 重构为一份“词典主手册 + 场景专题词库”的提示词编写参考表，既支持：

- 从零拼装 prompt
- 对已有 prompt 做漏项补全
- 从不同画面维度快速查词

## 非目标

首版不再把以下内容作为主体：

- 模型发布时间线
- API 调用与 SDK 安装
- 各家模型对比
- 大段原理解释
- 过多的完整长 prompt 示例

这些内容可以只保留极少量入口说明，不能再占据文档主体。

## 结构设计

### 1. 一眼入口

只说明这份文档的用途：

- 这是画面描述词典，不是模型介绍
- 适合查词、补词、拼装描述
- 从零写时先定主体，再按维度往下补

### 2. 最小拼装顺序卡

保留一张很短的装配顺序卡：

`主体 -> 场景 -> 构图 -> 镜头 -> 光线 -> 色彩 -> 材质细节 -> 氛围 -> 动作 -> 文字 -> 负面限制`

这部分只带路，不展开讲方法论。

### 3. 通用维度词典

作为文档主体，按以下维度展开：

- 主体
- 构图
- 镜头
- 光线
- 色彩
- 材质与细节
- 氛围
- 动作与状态
- 场景
- 风格
- 文字排版
- 负面限制

每个维度采用统一的卡片排法：

1. 一句话说明“这个维度在补什么”
2. 列出高频子类
3. 按子类铺可直接拼进 prompt 的中文短语
4. 放一个很短的组合提醒

重点不是做“同义词表”，而是做“能直接补画面的短语库”。

### 4. 场景专题词库

在通用维度词典之后追加高价值专题，首批包括：

- 人像
- 产品静物
- 电影感场景
- 海报与标题字
- 食物 / 饮品
- 室内空间
- 赛博 / 科幻
- 自然风光 / 天气

这些专题用于把高频场景里的关键词集中整理，避免只有通用维度没有实战组合。

### 5. 收尾卡

文末保留两类短卡：

- 常见缺词补全提醒
- 最常见错误

例如：

- 只有主体，没有空间关系
- 只有风格，没有材质细节
- 变量一次改太多
- 光线和情绪没有对齐

## 内容风格

- 中文为主
- 只有在中文容易歧义时，才在中文后补英文
- 少解释，多短语
- 候选词尽量可直接抄用
- 优先高密度、可扫描结构

## 风险与取舍

- 如果词库铺得太散，会变成“看起来很全，实际不好查”；因此必须先按高价值子类组织
- 如果专题过多，会稀释主词典；因此首版专题只保留高频 8 类
- 如果解释太多，会再次偏离“查词手册”；因此正文尽量不用大段说明
